AAA : Unit Testing

A unit test is a method that instantiates a small portion of an application and verifies its behavior independently from other parts.

Typical unit tests contains 3 phases:

- First, it initializes a small piece of an application it wants to test
- Then it applies some condition to the system under test
- Finally, it observes the resulting behavior


If the observed behavior is as expected, the unit test passes, otherwise, it fails, indicating that there is a problem somewhere in the system under test.

These three unit test phases are also known as Arrange, Act and Assert, or simply AAA.

Unit tests are designed to test state-based or interaction-based code logic:

Verifying that the system under test produces correct results, or that its resulting state is correct, is called state-based unit testing.
Verifying that it properly invokes certain methods is called interaction-based unit testing.


Test Method:

def test_calculator_subtract():
    """ In the Arrange phase, we create and set up a system under test.This can be empty and is not mandatory"""
    calc = Calculator()
    """The Act phase is where we poke the system under test, usually by invoking a method."""
    calc.subtract_number(1)
    """The Assert phase makes our unit test pass or fail."""
assert calc.get_result() == -1


Unit testing principles demand that a good test is:

- Easy to write
- Readable
- Reliable
- Fast
- Truly unit, not integration
